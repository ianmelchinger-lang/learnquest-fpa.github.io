Budget workflows

Set the exit criteria for a budget gate

When a company builds its yearly budget, each department fills in a template and hands it to finance. Before finance can add those pieces together, someone has to decide a submission is complete enough to move on. That checkpoint is a gate, and a gate only works if its rule for "complete" is one that two people, checking separately, would read the same way. Your job here is to tell those rules apart.

FP&A stands for financial planning and analysis, the team that plans and coordinates the budget. The trouble this activity is about is small but expensive: the word complete means different things to different people. A department head thinks a submission is done the moment it is sent; finance thinks it is done only when every field is filled and the assumptions are written down. A gate settles that argument, but only if its criteria can be checked the same way every time.

Budget cycle
The budget seen as a sequence of stages, not one event: set targets, send out templates, collect inputs, consolidate, review, revise, approve.
Consolidation
Adding every department's submitted template together into one company-wide budget.
Stage-gate
A checkpoint between two stages. Work moves past it only when set conditions are met.
Exit criterion
One condition a submission must meet to leave a stage. The gate is the full set of them.
Testable
A criterion is testable when it gives a plain yes or no. Two reviewers checking on their own reach the same answer.

Task

Sort the candidate criteria

Below are eight rules someone proposed for the gate that lets a department's budget template pass into consolidation. Some are testable, so they can hold every department to the same standard. Others sound fine but can't be enforced, because two reviewers could disagree on whether they are met. For each one, choose put on the gate or too vague to enforce. When you have sorted all eight, check them.

    Put on the gate

      Too vague to enforce

        The test is always the same: would two reviewers, checking on their own, always reach the same yes or no? A version number, a filled cell, a documented assumption, and a total that foots all pass that test. "Reasonable," "basically complete," "comfortable," and "roughly in line" do not, because each reviewer draws the line somewhere different. A gate built on those never actually stops anything.
